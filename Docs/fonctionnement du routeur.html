Solution envisagée router : 


définir des sections


chaque section est un niveau de la requête, les valeurs de sections peuvent être d'autres sections : 


exemple: 

[root]

coucou = "coucou.php"

eleves-polytechniciens = "x"


[x]

inscriptions = 'inscription'

test = "test.php"


[inscription]

enregistrement = "enregistrement.php" etc…


Le routeur utilise le cache dans lequel il place un tableau de type : 

array(

'url' => 'ficher à lire.php"

)


par exemple : array('x/connexion' => 'eleve.php');

Ce tableau est créé dans le cache s'il n'existe pas (ou si le cache n'est pas un vrai cache). De fait les choses vont plus vite et l'ini n'est lu qu'en cas de besoin.
